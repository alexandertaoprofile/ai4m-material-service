## 1) Repo 全量结构梳理文档（表格风格）

## 2) 风险分级清单（哪些模块不能先动）

### P0（高风险，禁先动）

| 对象 | 原因 | 允许动作 |
|---|---|---|
| `main.py` WebSocket 协议文本标记（如 `[start]` / `[end]` / Pending 语义） | 前端与客户端依赖强，轻微变动即可导致交互中断。 | 仅补注释与文档，不改字段和顺序。 |
| `/start`、`/uploadFile`、`/files`、`/roles` 接口输入输出结构 | 直接影响线上调用与前端解析。 | 仅做契约文档，不改行为。 |
| `src/team_config.py` 主流程分支与任务调度路径 | 业务核心链路，影响任务结果、资产组织、消息流。 | 仅做阅读梳理，不改逻辑。 |
| 产物路径与 URL 拼接规则（GLB/IMAGE/manifest） | 会影响前端资产访问与下载。 | 仅记录现状，不改规则。 |

### P1（中风险，谨慎后动）

| 对象 | 风险点 | 建议 |
|---|---|---|
| `config/config.yaml` 等配置键名 | 键名变更会触发运行时读取失败。 | 后续如改，先加向后兼容映射。 |
| `src/*_utils.py` 工具函数签名 | 上游调用点多，改动可能连锁影响。 | 重构前先做调用关系扫描。 |
| `alpha/provider/*` 模型适配 | 影响模型调用稳定性、超时重试与流式输出。 | 必须配套回归用例再动。 |

### P2（低风险，可先动）

| 对象 | 可做事项 |
|---|---|
| `docs/` 新文档 | 新增结构文档、运行说明、接口契约说明。 |
| 代码风格与静态检查配置文件 | `.editorconfig`、lint/format 配置（先不强制修复历史代码）。 |
| 开发辅助脚本（不入主链） | 仅用于检查、梳理、同步建议，不改业务逻辑。 |

## 3) Git 工作站改造实施清单（按低风险顺序，仅方案）

> 按你的要求：**暂不动现有 git 区**，此处只给执行顺序与落地标准。

### Step 1：先建立工程治理基础（零业务改动）
1. 新增治理文档：分支策略、提交流程、发布标签规则。  
2. 新增开发规范文件：`.editorconfig`、格式化/静态检查配置（先软约束）。  
3. 统一命令入口（如 `Makefile`）：`run` / `lint` / `smoke`。

**验收标准**：不改任何业务代码，服务行为完全一致。

### Step 2：建立“变更最小化提交”机制
1. 约定单 PR 单主题（文档/配置/代码分开）。  
2. 每次提交附“影响范围 + 回滚方式”。  
3. 对高风险目录要求双人复核（`main.py`、`src/team_config.py`）。

**验收标准**：提交记录可追踪、可回放、可回滚。

### Step 3：规范化改造从低风险区起步
1. 先改文档与脚本注释，再改格式问题。  
2. 最后才碰中风险模块；P0 模块排到末尾并需专项验证。  
3. 每个小步骤都对应 smoke test。

**验收标准**：任何时点都可回到上一稳定版本。

### Step 4：未来再处理 git 工作区形态
1. 评估“保持双目录同步”与“迁回单目录主仓”的成本。  
2. 选择迁移窗口，先演练再切换。  
3. 切换后保留一段并行观察期。

**验收标准**：开发流与发布流一致，不再依赖手工全量复制。

## 4) 不影响功能的验证清单（每一步怎么验）

### 4.1 启动与基础连通性

| 检查项 | 验证方式 | 通过标准 |
|---|---|---|
| 服务启动 | 按现有方式运行 `start.sh` 或 `python main.py` | 进程正常启动，无致命异常。 |
| 健康接口 | `GET /` | 返回 `{ "message": "XIMUAlpha_MNS server running." }`。 |

### 4.2 核心接口契约

| 检查项 | 验证方式 | 通过标准 |
|---|---|---|
| 角色接口 | `GET /roles` | 返回可解析角色列表，字段类型不变。 |
| 上传接口 | `POST /uploadFile`（含 `taskid` + files） | 文件落盘成功，返回成功消息。 |
| 文件列表接口 | `POST /files`（`taskid`） | 返回 `{files:[...]}` 且内容正确。 |

### 4.3 WebSocket 主流程

| 检查项 | 验证方式 | 通过标准 |
|---|---|---|
| `/start` 连接与初始化 | 发送 `idea/taskid/user_name/file_metadata` | 连接成功，流程开始。 |
| 消息标记兼容性 | 检查 `[start]`、`[end]`、`[Pending]` 等标记 | 标记文本与出现时序保持一致。 |
| 回合结束行为 | 完整跑完一轮 | 收到结束提示，连接关闭行为正常。 |

### 4.4 产物与路径一致性

| 检查项 | 验证方式 | 通过标准 |
|---|---|---|
| 输出文件结构 | 比较改造前后 task 产物目录 | 文件类型/命名/层级一致或兼容。 |
| URL 可访问性 | 抽查 GLB/图片链接 | 前端可正常访问，未出现 404。 |

### 4.5 回滚预案

| 场景 | 回滚动作 |
|---|---|
| 文档或配置变更导致问题 | 立即回退对应提交（单提交回滚）。 |
| 中风险模块变更异常 | 回退至上一个 smoke 通过点。 |
| 线上行为异常 | 恢复上一个发布 tag 并冻结新变更。 |
